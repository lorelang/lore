# Lorelang v0.2.1

Patch release focused on release hygiene and CI reliability.

## What changed

- Fixed CI failure in launch checks by ensuring `pytest` is installed in workflow setup.
- Bumped package/library version from `0.2.0` to `0.2.1`.
- Updated version-pinned tests to track `0.2.1`.
- Clarified release/versioning docs to distinguish package patch releases from language spec changes.

## Why this patch

`v0.2.0` was tagged before the CI dependency fix landed on `main`.
`v0.2.1` is the clean launch tag with passing launch checks.

## Validation

- `make launch-check` passed:
  - full test suite
  - conformance tests
  - validation of all example ontologies
  - compile matrix across built-in targets
  - end-to-end smoke workflow
